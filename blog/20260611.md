何が名詞で何が動詞で、だからここにくる、みたいな、英語ってかなり構造的な言語だよね、と思ってるんですが、SQLにはかなりそれを感じられる。
SELECT * FROM テーブル名 WHERE カラム名;  
SELECTは何言語？データ操作言語だね、CREATEは？データ定義言語、COMMITは？データ制御言語、みたいに、いちいち頭の中でもいいから「これは何をする言語である」を覚えておかないとなんというか身になりにくいのを感じてる。ということに今更気付いてほんとお前はいつも遅いんだよ！って感じだ。  


今日やったコード参照
```SQL
SHOW TABLES;

CREATE VIEW uriagedenwa
AS
SELECT idur,uriage.company,charge,salesdate,tel
FROM uriage
INNER JOIN jusho
ON uriage.company=jusho.company;
```
👆このコードがどういう内容なのかを分解する。  
> SHOWコマンドでまずテーブルを見る。  
> CREATEはデータ定義言語なのでデータを作るよ、じゃあ何作るの、VIEWでuriagedenwaを作るよ。  
> ASでuriagedenwaの中身はこれですと示す。  
> SELECTで列を指名し、  
> FROMどのテーブルから取得を決めて、  
> INNER JOINでuriageテーブルをjushoテーブルと結合するよ、  
> ON uriage.company = jusho.company  という条件で結合するよ。  


という内容。
